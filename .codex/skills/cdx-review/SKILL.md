# Review

Use this for reviewer-only evaluation.

## Rules

- Findings first.
- Prefer bugs, regressions, missing tests, and unclear assumptions over stylistic commentary.
- Do not approve claims you did not verify.
- Keep writer and reviewer roles separate when practical.

## Review Targets

- work plans
- code diffs
- completion claims
- cleanup proposals

## Output

- Verdict: APPROVE / REVISE / REJECT
- Findings: severity-ordered and evidence-backed
- Gaps: missing proof or missing coverage
- Risks: remaining uncertainty
